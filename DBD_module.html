Deciphering Big Data

Deciphering Big Data module will give us idea of the complex world of big data, exploring its diverse data types, sources, and collection methodologies.

Learning Objectives

  • Introduce and review various concepts of big data, technologies, and data management to enable you identify and manage challenges associated with security risks and limitations.
  • Critically analyse data wrangling problems and determine appropriate methodologies and tools in problem solving.
  • Explore different data types and formats. Evaluate various data storage formats ranging from structured, quasi structured, semi structured, and unstructured formats. We explore the various memory and storage requirements.
  • Critically examine various data collection methods and sources. Review fact finding methods to determine the integrity, reliability and readiness of data extracted and presented for pre-processing, cleaning, and usage.
  • Examine data exploration methods and analyse data for presentation in an organisation. Critically evaluate data readability, readiness, and longevity within the data Pipeline. Examine cloud services, API (Application Programming Interfaces) and how this enables data interoperability and connectivity.
  • Examine and analyse the ideas and theoretical concepts underlying DBMS (Database Management Systems) Database Design and Modelling.
  • Explore the future of use of data and deciphering by examining some fundamental ideas and concepts of machine learning and how these concepts are applied in various methods in handling big data.

Unit 6: Database Design and Normalisation

This unit provided a valuable deep dive into the world of database design and normalization. We explored the key concepts behind creating a relational database, focusing on how to structure and organize data efficiently.

    Here are some of the key takeaways from this unit:

  • Cleaning methods: We learned how cleaning methods play a crucial role in ensuring the quality and usability of datasets. This includes removing inconsistencies, duplicates, and errors from the data before storage.
  • Database creation and linking: We gained insights into the process of creating a database and how tables are linked using key fields. This establishes relationships between different data points, allowing for efficient retrieval and manipulation.
  • Anomalies and data integrity: The unit explored the concept of anomalies and how they can negatively impact data integrity. We learned how normalization helps to minimize these anomalies, ensuring data consistency and accuracy.
  • Normalization and normal forms: We delved into the concept of normalization and the different normal forms (1NF, 2NF, 3NF, etc.). Understanding the reasoning behind each form allows us to design databases that are efficient, maintainable, and less prone to data integrity issues.

Normalisation Task

Unnormalized Data

The table provided shows that there are repeating groups for Course Name, Exam Boards, and Teacher Name within the records of each student. Each of these groups is related but does not conform to the requirements of a relation in first normal form (1NF).

First Normal Form (1NF)

To convert to 1NF, we need to remove repeating groups and ensure that each field contains only atomic values:

  • Remove repeating groups: We must ensure that each field contains only one value.
  • Ensure each row is unique: Introduce a composite primary key if necessary, which could be a combination of Student Number and Course Name.
Resulting 1NF Table:

Second Normal Form (2NF)

To reach 2NF, we must remove partial dependencies of non-key attributes on the primary key.

Remove partial dependencies: Separate the data into multiple tables to ensure that each non-key attribute is fully functionally dependent on the primary key.

Third Normal Form (3NF)

In 3NF, we remove transitive dependencies; no non-key attribute should depend on another non-key attribute.

Remove transitive dependencies: If a non-key attribute depends on another non-key attribute, we create a separate table.

Each table now contains information without redundancy, and all attributes are dependent on the primary key.

Data Build Task

Introduction

With the normalization task completed, my focus shifted to constructing a relational database system that reflects the relationships and constraints inherent in the normalized data. This involved creating tables with designated primary and secondary (foreign) keys, establishing referential integrity to ensure data consistency.

Database Design

    Students Table
  • Student Number (Primary Key): A unique identifier for each student.
  • Student Name: The full name of the student.
  • Support: Indicator of whether the student receives support.
  • Date of Birth: The student’s date of birth.
    • Course Table
  • Course Name (Primary Key): A unique identifier for each course.
  • Exam Board: The exam board associated with the course.
    • Teachers Table
  • Teacher Name (Primary Key): A unique identifier for each teacher.
    • Course Enrollments Table
  • Enrollment ID (Primary Key): A unique identifier for each enrollment.
  • Student Number (Foreign Key): Links to the Students table.
  • Course Name (Foreign Key): Links to the Courses table.
  • Exam Score: The student's score for the particular course.
    • Course Teachers Table
  • Course Name (Foreign Key): Links to the Courses table.
  • Teacher Name (Foreign Key): Links to the Teachers table.
